 # FinTech 512 — Software Engineering Midterm Study Notes

## Overview

Comprehensive study notes for the FinTech 512 Software Engineering midterm exam, covering Weeks 1–6 of course materials. Notes are written in **Obsidian-flavored Markdown** with `[[internal links]]` for navigating between related concepts.

Notes are bilingual (English primary, Chinese supplementary) and designed to be beginner-friendly while covering all course-expected content.

## Notes Index

| File | Topics Covered |
|:---|:---|
| `Week1-Software-Engineering-Introduction.md` | SE fundamentals, SDLC models (Waterfall/Agile/Scrum/XP), Requirements Engineering (FR vs NFR), Quality Attributes, User Stories (INVEST), Use Cases, MVP, Git, TDD |
| `Week2-Software-Design.md` | Software Design principles, Modularity, Coupling & Cohesion, Information Hiding, UML Class Diagrams, Design Patterns intro, GRASP principles |
| `Week3-Reviews-OODesign-Refactoring.md` | Code Reviews, OO Design (SOLID), Module Decomposition, Code Smells catalog, Refactoring techniques (Extract Method, Extract Class, etc.) |
| `Week4-DesignPatterns-Testing.md` | GoF Design Patterns (Observer, Strategy, Singleton, Factory, etc.), Unit Testing, pytest, Code Coverage (Statement/Branch/Path) |
| `Week5-Databases-APIs.md` | Relational Databases, Database Normalization (1NF–3NF), SQL (DDL/DML), SQLAlchemy, Transactions, Web APIs & REST, AJAX |
| `Week6-TestDoubles-Methodologies.md` | Test Doubles (Dummy/Stub/Spy/Mock/Fake), unittest.mock, Dependency Injection, Agile/Scrum/Kanban/XP/Lean, Estimation (Story Points, COCOMO, PERT), UI/UX Design, Nielsen's Heuristics |
| `Midterm-Exam-Preparation.md` | Exam-focused guide based on 2023 & 2024 past midterms — topic mapping, priority study plan, practice questions with solutions, cheat sheets |

## How to Use

1. Open the folder in [Obsidian](https://obsidian.md/)
2. Start with `Midterm-Exam-Preparation.md` for exam strategy and priority topics
3. Follow `[[links]]` to jump to specific concepts in the weekly notes
4. Use the "Exam Quick Review" section at the bottom of each weekly note for fast revision
